# 🚀 泡沫雷达 — 完整部署指南

> 前端已构建成功，现在帮你完成部署。整个流程分为两步：前端部署 + 后端部署。

---

## 📋 部署前准备

### 你需要准备的东西

| 项目 | 用途 | 是否必须 |
|------|------|----------|
| Cloudflare 账号 | 部署前端（免费） | ✅ 必须 |
| 国内服务器（腾讯云/阿里云轻量） | 部署后端 | 可选，但建议有 |
| API Keys（FRED、Alpha Vantage、OpenAI） | 后端数据拉取 | 可选 |
| 域名（可选） | 自定义访问地址 | 可选 |

> **省钱方案**：Cloudflare 前端免费 + 腾讯云轻量服务器（约50元/月，新用户首年更便宜）

## 🌐 第一步：部署前端到 Cloudflare Pages

### 方式 A：拖拽上传（最简单，推荐）

1. **注册 Cloudflare 账号**
   - 访问 https://dash.cloudflare.com/sign-up
   - 用邮箱注册，验证邮箱

2. **进入 Pages 控制台**
   - 登录后点击左侧菜单 **Workers & Pages**
   - 点击 **Create**
   - 选择 **Pages** → **Upload assets**

3. **上传构建产物**
   - 在弹出的窗口中，**拖拽 `frontend/dist` 文件夹**进去
   - 或者点击上传按钮选择 `frontend/dist` 文件夹
   - 等待上传完成（约1-2分钟）

4. **获取访问地址**
   - 上传完成后，Cloudflare 会给你一个类似 `https://xxx.pages.dev` 的地址
   - 这就是你的前端网址！可以直接访问，也可以分享给用户安装 PWA

5. **自定义域名（可选）**
   - 在 Pages 设置中添加自定义域名
   - 按提示修改 DNS 记录即可

### 方式 B：Wrangler CLI 命令行部署（适合后续更新）

```bash
# 1. 安装 Wrangler CLI
npm i -g wrangler

# 2. 登录 Cloudflare
wrangler login
# 会打开浏览器让你授权，点击"允许"

# 3. 部署（在 dist 目录中执行）
cd frontend/dist
wrangler pages deploy . --project-name=bubbleradar
```

后续更新只需要重新构建后再次执行第 3 步。

## 🔧 第二步：部署后端 API（需要服务器）

后端是 Python FastAPI，需要服务器运行。推荐以下方案：

### 方案 1：国内服务器（推荐，访问快）

**推荐平台**：腾讯云轻量服务器 / 阿里云轻量服务器
- 配置：2核2G 即可
- 价格：约50元/月（新用户首年可能更便宜）
- 系统：Ubuntu 22.04

**部署步骤**：

1. 购买服务器后，用 SSH 连接

```bash
ssh root@你的服务器IP
```

2. 安装 Docker 和 Docker Compose

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 安装 Docker Compose
sudo apt-get update
sudo apt-get install -y docker-compose-plugin
```

3. 上传项目到服务器

```bash
# 在本地电脑执行，将项目传到服务器
scp -r C:/Users/19029/Desktop/郎中网站/金融网站 root@你的服务器IP:/opt/bubbleradar
```

4. 配置环境变量

```bash
cd /opt/bubbleradar
cp .env.example .env
nano .env
# 修改以下关键配置：
# - DATABASE_URL（使用 SQLite 最简单）
# - SECRET_KEY（改成一个随机字符串）
# - CORS_ORIGINS=["https://你的前端地址.pages.dev"] （重要！填写你前端的真实地址）
# - API Keys（如果有的话）
```

5. 启动后端

```bash
# 使用 Docker Compose 启动
docker-compose up -d

# 查看日志
docker-compose logs -f backend
```

6. 获取后端地址

- 后端默认运行在服务器的 8000 端口
- 你需要在服务器安全组中放行 8000 端口
- 或者通过 Nginx 反向代理到 80/443 端口

**后端地址示例**：`https://api.bubbleradar.com` 或 `http://你的服务器IP:8000`

### 方案 2：免费方案（Render / Railway）

如果你没有服务器，可以用国外的免费平台：

- **Render**：https://render.com（免费，但国内访问慢）
- **Railway**：https://railway.app（有免费额度）

但**强烈不建议用于国内用户**，访问速度会很慢。

## 🔗 第三步：前端连接后端

后端部署完成后，你需要重新构建前端，让前端知道后端地址在哪里。

### 方法：重新构建前端（传入后端地址）

1. 修改前端构建命令

打开项目根目录下的 `deploy-frontend.bat`，运行它：

```bash
# Windows 双击运行
deploy-frontend.bat

# 输入你的后端地址，例如：
# https://api.bubbleradar.com
# 或者 http://你的服务器IP:8000
```

2. 构建完成后，重新上传到 Cloudflare Pages

   - 拖拽新的 `frontend/dist` 文件夹上传
   - 或运行 `wrangler pages deploy .`

3. 完成后，前端就能正确访问后端 API 了

以后代码修改后，按这个顺序更新：

```bash

# 2. 重新构建并上传前端
deploy-frontend.bat
```

## ⚠️ 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 前端页面能打开，但数据加载失败 | 后端地址没配置对 | 检查 CORS_ORIGINS 和前端 VITE_API_BASE_URL |
| 后端启动失败 | 环境变量没配置 | 检查 .env 文件中的数据库路径、API Keys |
| 手机安装 PWA 后无法访问 | 后端服务没启动 | 确认后端服务器在运行 |
| 上传 Cloudflare 后页面空白 | 构建产物路径错误 | 确保上传的是 `frontend/dist` 文件夹 |
| 跨域错误（CORS） | 前后端域名不匹配 | 后端 CORS_ORIGINS 必须包含前端域名 |
